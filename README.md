# Бот по заявкам

Есть таблица запусков в Excel — с объектами, заказами, датами и цифрами. Хочется не копировать это вручную в письмо, не рисовать скриншоты и не терять нитку, когда объектов несколько, а заказчик ждёт и письмо, и картинку, и файл, который можно открыть в Excel. Эта программа как раз об этом: она подключается к вашему файлу, разбирает нужные листы, собирает аккуратное «примечание» для письма и по одному нажатию отправляет материалы получателям — на e-mail и в Telegram.

Ниже — что она умеет, без занудства, но чтобы ничего важного не потерять.

---

## Что это за программа

Локальное приложение для **Windows** с графическим окном. Работает **на вашем компьютере**: Excel вы выбираете сами, данные никуда «в облако» сами по себе не уходят — только туда, куда вы настроите почту и Telegram.

---

## Главные возможности

### Excel: подключить, обновить, не бояться объёма

- Указываете путь к файлу вручную или через кнопку **«Обзор…»**, затем **«Подключить файл»** (или **«Обновить данные из Excel»**, когда таблица уже открыта и вы её поправили).
- Поддерживаются форматы **`.xlsx`**, **`.xlsm`**, **`.xls`**.
- Парсер ориентируется на структуру ваших рабочих таблиц (листы в духе жилых комплексов, якорь **«АСТРА»** и т.д.) — читается не весь файл подряд, а та часть, что заложена в логике разбора.
- После успешного подключения путь к файлу **сохраняется** в `settings.json` — в следующий раз можно не искать файл заново.
- Рядом с программой ведётся **локальный кэш** (`cache.db`): так проще отслеживать, что в таблице изменилось с прошлого раза (новые, изменённые, удалённые записи — для внутренней сверки).

### Объекты и заказы: выбрать точечно или «всё сразу»

- Выпадающий список **«Объект»** — конкретный объект или пункт **«Все объекты»**.
- Список **«Наименование заказа»** с **мультивыбором** (галочки): можно отметить несколько заказов.
- Если нужно собрать в **одно** письмо данные **с нескольких объектов**, переключайтесь между объектами и отмечайте заказы по очереди: выбор **запоминается по каждому объекту** и не сбрасывается при переключении и в режиме «Все объекты» (одинаковые номера заказов на разных объектах **не затирают** друг друга).

### Текст письма и предпросмотр

- Поле **«Текст письма / примечание»** заполняется **автоматически** из выбранных строк — в виде ровных колонок, удобных для чтения в письме.
- Перед отправкой текст **можно отредактировать** вручную — то, что в поле, и уйдёт получателю.

### Сотрудники: список, кто получит письмо или Telegram

- Список **«Сотрудники (мультивыбор)»** — из файла `employees.json`.
- Для каждого: **ФИО**, **e-mail**, **ID Telegram** (личный или групповой чат — как заведёте в записи).
- Можно **добавлять** новых, **удалять** выбранных, **править** выбранную строку (с записью обратно в `employees.json`) — без программирования, из окна программы.
- В списке предусмотрены визуальные разделители (например, групповой чат и отдельные люди) — чтобы не путаться в длинном перечне.

### Дополнительные вложения

- Область **перетаскивания файлов**: можно добавить к отправке любые нужные документы (сметы, сканы, доп. таблицы). Список вложений виден до отправки.

### Отправка на e-mail

- Настройка **SMTP** в `settings.json`: сервер, порт, логин, пароль, режим **SSL / STARTTLS / без шифрования**.
- Поле **«E-mail получателя»**:
  - если в списке сотрудников **никто не выделен** — можно ввести адрес заказчика вручную или оставить пустым и использовать адрес по умолчанию из настроек;
  - если **выделены сотрудники** — письма уходят на **их** e-mail из `employees.json` (поле ввода в этом случае не используется).
- В одной отправке формируется **набор вложений по выбранным данным**:
  - по каждому объекту в выборке — **PNG-картинка** (наглядная выкопировка) и **фрагмент в формате Excel** (`.xlsx`) для тех же строк.
- Тема письма человеку формируется с учётом того, один объект в выборке или несколько.

### Отправка в Telegram

- Кнопка **«Отправить в Telegram»**: документы (с тем же смыслом, что и для почты — сгенерированные файлы по выбору + ваши доп. вложения) и подпись к сообщению.
- Если сотрудники **не выделены** — используется **общий** `telegram_chat_id` из `settings.json`.
- Если выделены — сообщения уходят в чаты, указанные в **`telegram_chat_id`** у каждого в `employees.json` (удобно и для личных, и для группового чата как отдельной «строки» в списке).
- Токен бота задаётся в `settings.json`.

### После успешной отправки

- Чтобы случайно не отправить **то же самое** второй раз подряд, выбор заказов и дополнительные вложения **сбрасываются**; нужно снова отметить заказы и при необходимости добавить файлы.

### Сеть и диагностика

- Перед отправкой проверяется **доступность сети** (DNS): если интернет или доступ к сервисам «режется», программа предупредит, а не будет молча зависать.

### Журнал

- Работа приложения пишется в **`app.log`** рядом с exe или скриптом — удобно, если нужно разобрать «что пошло не так».

## Файлы рядом с программой

Рабочая папка — там, где лежит **`zayavki_run.exe`** или **`zayavki_run.py`** (не внутри `zayavki_bot_app`).

| Файл | Зачем |
|------|--------|
| `settings.json` | Почта, Telegram, последний путь к Excel |
| `employees.json` | Список сотрудников |
| `cache.db` | Кэш и сверка изменений (создаётся сам) |
| `app.log` | Журнал (появляется при работе) |

Если `settings.json` или `employees.json` **нет** при первом запуске собранной программы, подтянутся **встроенные шаблоны** из сборки — дальше вы сохраняете свои настройки через интерфейс или правите JSON.

## Запуск из исходников

Из **корня проекта** (где лежит `zayavki_run.py`):

```bash
pip install -r zayavki_bot_app/requirements.txt
python zayavki_run.py
```

Для сборки exe дополнительно: `pip install pyinstaller`.

---

## Сборка `.exe` (PyInstaller, режим one-folder)

Из корня проекта:

```bash
py -3 -m PyInstaller zayavki_run_onedir.spec --clean --noconfirm
```

Готовая папка: **`dist\zayavki_run\`** — её целиком переносите на другой компьютер (обязательно вместе с **`_internal`**). Запуск: **`zayavki_run.exe`**.

---

## Автотесты (для разработчика)

Проверка логики выбора заказов по объектам (без окна программы):

```bash
py -3 -m unittest tests.test_order_selection_logic -v
```
